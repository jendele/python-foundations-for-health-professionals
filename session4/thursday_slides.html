Day 2
Part 1

Comprehensions, deeper.

"For every X in Y, if Z, do W" — in one expression.

Comprehensions
§1 · Anatomy recap

Three building blocks. One bracket.

[ output     for    item     in    source     if    condition ]
     ↑                ↑              ↑                  ↑
  what comes      where each      where do          which ones
  out per item    item comes      you get them      do you keep?

Read aloud: "For every item in the source, if the condition holds, produce the output."

The bracket determines the data structure: [] list · {} set · {k: v} dict

The structure that carries everything
§1.2 · List comprehension

Transform & filter.

# All ages
all_ages = [p["age"] for p in patients]

# Only patients aged 65+
elderly = [p["id"]
           for p in patients
           if p["age"] >= 65]

# Combined: cardiology AND seniors
cardio_seniors = [
    p["id"]
    for p in patients
    if p["ward"] == "Cardiology"
    and p["age"] >= 65
]

Each line of the comprehension answers one question:

  • What — the expression at the start
  • Where from — the for-clause
  • Which ones — the if-clause

Use AND when both conditions must hold. Use OR when either is enough.

Filter, transform, both
§1.3 · Set comprehension

Same shape. Curly braces. Duplicates removed.

# With duplicates — list comp
wards_list = [p["ward"] for p in patients]
# ['Cardiology','InternalMedicine','Geriatrics',
#  'Surgery','Cardiology','Geriatrics',...]

# Unique only — set comp
wards_set = {p["ward"] for p in patients}
# {'Cardiology','InternalMedicine','Geriatrics',
#  'Surgery','ICU'}

The bracket changes everything.

When the answer to "how many distinct values?" matters, use a set — comprehension or otherwise.

Same logic, different container
§1.4 · Dict comprehension

Build a lookup table.

# Patient ID → age
age_lookup = {p["id"]: p["age"]
              for p in patients}

age_lookup["P0003"]   # 82 — instant
age_lookup["P0042"]   # KeyError

# Safer access:
age_lookup.get("P0042", "n/a")  # 'n/a'

Once you have a lookup dict, finding any patient's age is O(1) — instant, regardless of dataset size.

With a list of dicts, you'd have to scan everyone. With a lookup dict, one operation.

The dict comprehension builds your indexes
§1.5 · The limit

When NOT to comprehension.

Use a regular for-loop when:

  • Multiple steps per item
  • Side effects (printing, plotting)
  • The line gets longer than ~80 chars
  • You're nesting comprehensions

Comprehensions are a readability tool, not a status symbol. If you can't read it on one line, it's the wrong shape.

# Don't do this — unreadable
[f"{p['id']}: {' & '.join(p['chronic'])
 if p['chronic'] else 'no comorbidities'}"
 for p in patients
 if p["age"] >= 65 and (p["ward"] in
   ["Cardiology", "Geriatrics"]
   or len(p["chronic"]) >= 2)]

# Better — readable loop with named steps
result = []
for p in patients:
    is_old = p["age"] >= 65
    is_cardio_or_geriatr = p["ward"] in [...]
    has_many = len(p["chronic"]) >= 2
    if is_old and (is_cardio_or_geriatr or has_many):
        ...
Choose readability
Part 2

Functions, deeper.

A function is a reusable rule. Today we design good ones.

Functions
§2.1 · One function, one job

Each function does one thing well.

def average_age(patient_list):
    """Return the mean age of patients."""
    if not patient_list:
        return 0
    ages = [p["age"] for p in patient_list]
    return sum(ages) / len(ages)

# Reuse on any subset — function doesn't care how it was built
average_age(patients)
average_age([p for p in patients if p["ward"] == "Cardiology"])
average_age([])   # 0 — handled gracefully

A clinical protocol — defined inputs, defined output, robust against edge cases.

The smallest reusable unit
§2.2 · Default arguments

Flexible without forcing every caller to specify everything.

def filter_by_age(patient_list, min_age=65, max_age=None):
    """Filter patients by age range."""
    result = [p for p in patient_list if p["age"] >= min_age]
    if max_age is not None:
        result = [p for p in result if p["age"] <= max_age]
    return result

filter_by_age(patients)                              # defaults: 65+
filter_by_age(patients, min_age=50)                  # 50+
filter_by_age(patients, min_age=65, max_age=75)      # 65-75

Important: default arguments must be immutable. Use None, not [] or {}.

Smart defaults make APIs friendly
§2.3 · Returning more than one thing

Use a tuple. Caller unpacks.

def age_stats(patient_list):
    """Return (min, max, mean) ages."""
    ages = [p["age"] for p in patient_list]
    return min(ages), max(ages), sum(ages) / len(ages)

# Tuple unpacking on the call
youngest, oldest, average = age_stats(patients)

# Or capture as one tuple
stats = age_stats(patients)
print(stats)   # (39, 90, 65.7)

Multi-return is just one tuple — Python lets the caller decide whether to unpack or keep it as a tuple.

One return value can be a tuple
§2.4 · Composition

Build small. Combine. That's the trick.

def is_high_risk(patient, age_threshold=65, min_conditions=2):
    return (patient["age"] >= age_threshold
            and len(patient["chronic"]) >= min_conditions)

def high_risk_patients(patient_list):
    return [p for p in patient_list if is_high_risk(p)]

def report(patient_list):
    hr = high_risk_patients(patient_list)
    print(f"Total: {len(patient_list)}, high-risk: {len(hr)}")

If the rule for "high-risk" changes tomorrow, you change one function. Everything else keeps working. That's modularity.

Functions that call functions
§2.5 · Lambda

The tiny one-shot function.

# Sort patients by age
sorted(patients, key=lambda p: p["age"])

# Find patient with most chronic conditions
max(patients, key=lambda p: len(p["chronic"]))

# Sort by ward, then by age descending within ward
sorted(patients, key=lambda p: (p["ward"], -p["age"]))

Lambda is for tiny, one-shot, in-line functions. Use it as a key= argument and similar.

If your lambda has any logic beyond "extract this field" — promote it to a real def.

The right tool, in moderation
Part 3

Scope.
Where do variables live?

L · E · G · B

Scope
§3.1 · The LEGB rule

Python looks up names in four places — in this order.

  • L — Local (current function)
  • E — Enclosing (surrounding function)
  • G — Global (module level)
  • B — Built-in (Python's own)

Whoever is found first wins. The lookup stops there.

x = 100                          # Global

def outer():
    x = 50                       # Enclosing

    def inner():
        x = 5                    # Local
        print(x)                 # 5

    inner()
    print(x)                     # 50

outer()
print(x)                         # 100
Names are resolved by location
§3.2 · Local stays local

What happens inside, stays inside.

def compute_score(patient):
    bonus = 10
    score = patient["age"] + len(patient["chronic"]) * 5 + bonus
    return score

risk = compute_score(patients[0])
print(risk)              # 87

print(bonus)             # NameError
                         # bonus was local — it's gone

Variables inside a function are destroyed when the function returns. The only way to bring a value out is to return it — and the caller must catch it with =.

Functions are sealed boxes
§3.3 · Two safe rules

Follow these. Most scope bugs disappear.

Rule 1

Always return what you compute.

Don't rely on side effects. Predictable, testable, no surprises.

Rule 2

Don't read or write globals from inside a function.

Pass them as parameters. Functions become self-contained units.

Following these turns 80% of beginner bugs into non-issues.

Discipline, not memorisation
Wrap-up

The three patterns that come up over and over.

# 1. Filter (comprehension)
subset = [item for item in collection if condition]

# 2. Transform (comprehension)
new_collection = [f(item) for item in collection]

# 3. Aggregate (function)
def summary(collection):
    return some_metric_calculated_from(collection)

Combine them: filter → transform → aggregate. That's the workflow of every analysis you'll do for the rest of the course.
