Session 2 — Logic
Phase 2 · Block C

The decision block: if / elif / else

if systolic >= 180:
    category = "hypertensive crisis"
elif systolic >= 140:
    category = "hypertension"
elif systolic >= 130:
    category = "elevated"
else:
    category = "normal"

Reads top-to-bottom

The first branch whose condition is True wins. All others are skipped.

The else is the catch-all

It runs when no prior condition matched. Use it for "everything else" — not as a fallback for things you forgot.

Session 2 — if/elif/else
Phase 2 · Block C

In Python, indentation is code

if systolic >= 140:
    print("flag patient")
    notify_clinician()

# two spaces / tabs different
# → IndentationError

What the indent means

"These lines belong to the if-block." The indented block is what runs when the condition is True.

Rule: 4 spaces. Never mix tabs and spaces. VSCode does this for you.

Why Python does this: to force readable code. In other languages you can write a 200-line mess that runs; Python refuses to run what it cannot visually parse.

Session 2 — Indentation
Phase 2 · Block D

The for loop — "do this for each …"

patients = [patient_1, patient_2, patient_3]

for p in patients:
    bmi = p["weight_kg"] / (p["height_m"] ** 2)
    print(f'{p["name"]}: BMI = {bmi:.1f}')

The loop variable p

Takes one value from the list per iteration. Name it after what it represents, not x or i.

This is the payoff

Three patients, three lines of output — but one block of code. 30 patients? Same block. 3 million? Same block.

Session 2 — for
Phase 2 · Block D

The while loop — "keep going while …"

attempts = 0
while attempts < 3:
    answer = input("Enter a valid ID: ")
    if answer.startswith("P"):
        break
    attempts += 1

for vs. while

for = iterate a known collection. while = repeat until some condition becomes False. Use for when you can.

Danger: infinite loop

If the condition never becomes False, the loop runs forever. Always make sure something inside the loop changes the condition.

Session 2 — while
Phase 2 · Block D

break and continue

for p in patients:
    if p["age"] < 18:
        continue   # skip this one
    process(p)

continue → skip the rest of this iteration, go to the next.

for p in patients:
    if p["id"] == target_id:
        found = p
        break      # stop searching

break → exit the loop entirely.

Use sparingly. Most loops don't need them. When they do, they make the intent crystal clear: "this patient is out" or "I found what I needed, stop."

Session 2 — break/continue
Phase 2 · Bonus

List comprehension — a sneak peek

# Long form
bmis = []
for p in patients:
    bmis.append(p["weight_kg"] / (p["height_m"] ** 2))

# Short form — a list comprehension
bmis = [p["weight_kg"] / (p["height_m"] ** 2) for p in patients]

Same result, one line. Reads as: "make a list of BMI for each p in patients." Use it when the transformation fits on one line. Skip it when the logic gets nested — a plain for loop is always more readable.

Session 2 — Phase 3
Phase 3 · Behaviour 1

Truthiness — not every condition needs ==

Falsy values

0              # zero int
0.0            # zero float
""             # empty string
[]             # empty list
{}             # empty dict
None           # the "nothing" value
False          # literal

Everything else is truthy

if patient_list:
    print("we have patients")
# runs iff list is non-empty

if name:
    print(name)
# runs iff name is not ""

Use truthiness for "is there anything here?" checks. Use == when comparing against specific values.

Session 2 — Truthiness
Phase 3 · Behaviour 2

Short-circuit evaluation

Python stops evaluating as soon as the answer is certain.

# With `and`: if the first is False, the rest is irrelevant
if patient and patient["age"] > 65:
    ...
# If patient is None, the second part is never evaluated — no crash.

# With `or`: if the first is True, the rest is irrelevant
display_name = name or "unknown"
# Uses `name` if truthy, falls back to "unknown" otherwise.

Clinical parallel: a red-flag checklist stops as soon as one red flag is found. Python's or does exactly that. A criteria-all-must-hold list stops as soon as one fails. That's and.

Session 2 — Short-circuit
Phase 3 · Behaviour 3

What really happens in a for loop

Python asks the list: "give me your iterator"
↓
Iterator hands out one element at a time, on request
↓  each next() call
Body of the loop runs with the loop variable bound to that element
↓  repeat until iterator is exhausted
Loop ends · the loop variable keeps its last value

The iterator is the reason for works the same for lists, strings, dicts, files, and huge datasets that do not fit in memory.

Session 2 — BP reference
Phase 4 · Pattern

The classification pattern

def classify_bp(systolic, diastolic):
    if systolic >= 180 or diastolic >= 110:
        return "Grade 3"
    elif systolic >= 160 or diastolic >= 100:
        return "Grade 2"
    elif systolic >= 140 or diastolic >= 90:
        return "Grade 1"
    elif systolic >= 130 or diastolic >= 85:
        return "High-normal"
    else:
        return "Normal"

for p in patients:
    p["bp_category"] = classify_bp(p["systolic"], p["diastolic"])

Functions come formally in Session 3 — but you already have the intuition: "a mini-procedure with a name."

Session 2 — Pattern
Phase 5 · Differential Diagnosis

Four bugs you will meet today

Name them now. Recognise them when they bite.

1 · = vs. ==

if age = 65: → SyntaxError. if age == 65: → the comparison you meant.

2 · Off-by-one in ranges

range(1, 10) goes 1–9, not 1–10. Python ranges are exclusive on the right. Trip-wire for everyone coming from 1-indexed systems.

3 · Infinite while

The condition never becomes False. Tab closes but kernel stays busy → restart the kernel to free it.

4 · Wrong indentation

One line indented too much or too little changes which if it belongs to. Python will either refuse to run it or run it wrong but silently.
