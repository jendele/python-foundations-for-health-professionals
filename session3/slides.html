Session 3 · Block B · Day 1
§1 · Why structures matter

A patient is more than one number.

Every admission has many facets:

  • Who — id, age, gender, insurance
  • Where — ward
  • When — admission & discharge
  • Why — DRG code, diagnosis
  • How much — cost in EUR

Parallel lists work — but they hide meaning in position. Add one column and everything else must stay in sync.

# The naive "parallel lists" approach
patient_ids = ["P0001", "P0002", "P0003"]
ages        = [67, 54, 82]
wards       = ["Cardio", "IM", "Geriatrics"]
los_days    = [5, 3, 9]

# Find age of P0002 — count, don't read.
i = patient_ids.index("P0002")
print(ages[i])  # 54
Why we need structures
§2 · List & Tuple

Ordered. Indexed. Mutable or sealed.

list [ ]

Ordered, mutable. A queue you can edit.

admissions = ["09:14", "10:02"]
admissions.append("10:47")

tuple ( )

Ordered, immutable. A sealed envelope.

location = (52.3759, 9.7320)
# location[0] = 53.0  → TypeError

Tuples are hashable — they can be dict keys or set elements. Lists cannot.

Choose the shape that signals intent
§3 · Dict

Keyed access. The natural shape of a record.

Maps keys (meaning) to values (data).

One dict = one encounter. Read for meaning, not position.

encounter["ward"] reads better than encounter[3].

Use .get(key, default) for safe access when keys may be missing.

encounter = {
    "encounter_id":  "E00042",
    "patient_id":    "P0001",
    "ward":          "Cardiology",
    "drg_code":      "F62B",
    "los_days":      5,
    "actual_cost_eur": 3450.00,
}

print(encounter["ward"])           # Cardiology
encounter.get("nurse", "tba")      # 'tba'
The most useful structure in clinical data
§4 · Set

Uniqueness. Fast membership.

Unordered, no duplicates, hashable elements only.

Mental model: a whiteboard list where you only care whether something is on it.

  • a & b — intersection
  • a | b — union
  • a - b — difference
wards_today = ["Cardio","ICU","Cardio","Surgery","ICU"]
distinct = set(wards_today)
# {'ICU','Cardio','Surgery'}

"ICU" in distinct          # True

yesterday = {"ICU","Geriatrics","Pulmonology"}
both = distinct & yesterday   # {'ICU'}
Use for de-duplication and membership

§5 · Iteration & list comprehensions

Say what you want, not how to loop.

# Classic loop — works:
total = 0
for e in encounters:
    total += e["actual_cost_eur"]
print(total)
# Same, comprehension form — read aloud:
costs = [e["actual_cost_eur"] for e in encounters]
print(sum(costs))

# With a filter:
cardio = [e["actual_cost_eur"] for e in encounters
          if e["ward"] == "Cardiology"]

Read it: "for every encounter e in encounters, take e[\"actual_cost_eur\"]."

The pythonic loop

§6 · Functions

Named code. Reusable rules.

A function takes parameters, does something, returns one value.

Think of it as a reusable clinical rule: "Given a list of encounters, return mean LOS." Once written, you trust it.

Always include a one-line docstring — your future self will thank you.

def mean_los(encounter_list):
    """Mean length of stay across encounters."""
    if not encounter_list:
        return 0
    total = sum(e["los_days"] for e in encounter_list)
    return total / len(encounter_list)

mean_los(encounters)            # 5.6

cardio = [e for e in encounters
          if e["ward"]=="Cardiology"]
mean_los(cardio)                # 4.5
The smallest reusable unit
§7 · Three traps

Scope · silent None · mutable defaults.

Scope

def f(x):
    y = x*2
    return y

f(3)        # 6
print(y)    # NameError

Silent None

def summary(e):
    print(e["los_days"])
    # no return!

r = summary(e1)
r           # None

Mutable default

def bad(msg, log=[]):
    log.append(msg)
    return log

bad("a")    # ['a']
bad("b")    # ['a','b'] !

Fix the third one with def good(msg, log=None): if log is None: log = []

§8 · Clinic

Bed Occupancy Calculator.

The CMO's question, end-to-end.

Logic, in three sentences:

  1. For a date d, find encounters where admission ≤ d < discharge.
  2. Group those by ward — count = beds used.
  3. Divide by capacity → occupancy rate.

Three sentences · three small functions · one KPI.

The keystone of the day
§8 · Three small functions

One job each. Trust the interfaces.

def is_in_house(encounter, target_date):
    """True if patient was inpatient on target_date."""
    adm = date.fromisoformat(encounter["admission_date"])
    dc  = date.fromisoformat(encounter["discharge_date"])
    return adm <= target_date < dc

def beds_used_per_ward(encounters_list, target_date):
    """{ward: count} for that date."""
    counts = {}
    for e in encounters_list:
        if is_in_house(e, target_date):
            counts[e["ward"]] = counts.get(e["ward"], 0) + 1
    return counts

def occupancy_rate(encounters_list, target_date, capacity=WARD_CAPACITY):
    """{ward: rate_0_to_1} for that date."""
    used = beds_used_per_ward(encounters_list, target_date)
    return {w: used.get(w, 0) / cap for w, cap in capacity.items()}
Modular thinking — debug one piece, not the whole script
