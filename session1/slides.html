Session 1 — GUI vs Terminal
Phase 2 · Block B

Jupyter = Server + Kernel(s)

Jupyter Server

One process that manages everything: the file browser, the web UI, the routing. One server for your whole session.

Clinic admin — the front desk.

Kernel

A separate Python process, one per notebook. Runs the code. Holds all variables in RAM until it dies.

Individual exam room — one per patient.

Five open notebooks = 1 server + 5 kernels.
Close the tab → the kernel keeps running. Tab closed ≠ room cleaned.

Session 1 — GDPR
Phase 2 · Block D

Git ≠ GitHub

Git is a version-control program. Runs on your laptop. Works offline.
GitHub is a website that hosts Git repositories.

my_project/
├── .git/          # all version history lives here
├── analysis.py
└── patients.csv

Every commit

Like an entry in a patient chart: who · when · what. Nothing overwrites — everything is added.

.gitignore

A file that tells Git: "these files never enter the repo." The key for data stays.

Session 1 — Path 1
Phase 3 · Path 2

I run git clone

GitHub server (somewhere on the internet)
↓  git clone https://…
Your laptop's disk · full copy including .git/ history
↓  cd into the folder
You can open · edit · run — all locally, offline

Why this is so useful

One command = the whole project and its history on your machine. You can change things without touching the original — until you push.

Session 1 — Path 2
Phase 3 · Path 3

I run pip install pandas

Your terminal: pip install pandas
↓
pip contacts the Python Package Index (pypi.org)
↓  downloads & installs
Your laptop's Python environment · pandas now available
↓
Any notebook can now import pandas

Libraries are not part of Python. They are installed alongside it. Each virtual environment needs its own install.

Session 1 — Path 3
Phase 3 · Path 4

I close the browser tab

Tab closes · browser disconnects
↓  but on your laptop…
Jupyter server: still running
Kernel: still alive in memory
Variables: still there, still eating RAM

Closing the door of the exam room does not mean the room is cleaned.
Use File → Close and Halt, or shut the kernel down from the Running panel.

Phase 5 · Differential Diagnosis

Four kinds of weird you will meet

Name them now. Recognise them later.

1 · Hidden state

Cell 3 uses a variable cell 1 defined. You changed cell 1 and didn't re-run. Cell 3 uses the old version.

Fix: Restart kernel · Run All.

2 · Tab closed, RAM still full

You closed the notebook. The kernel is still alive. RAM is still held.

Fix: File → Close and Halt.

3 · Why doesn't pip work in my notebook?

Because pip is a shell command, not Python. It belongs in the terminal.

Shortcut: !pip install … works in cells — but blurs the hierarchy.

4 · Where did my DataFrame go?

You restarted the kernel. Everything in RAM was freed. Notebook file is intact — your live data is not.

Fix: re-run all cells from the top.
